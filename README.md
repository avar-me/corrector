# corrector.avar.me

Аварский корректор — веб-инструмент для проверки орфографии и исправления текстов после OCR.

## Возможности

- **Проверка слова** — введите слово и получите список вариантов с вероятностью, расстоянием редактирования и частотой
- **Исправление текста** — введите текст и получите исправленную версию с подсвеченными изменениями
- **OCR-режим** — применяет дополнительные правила замены символов (например, П → ГӀ)

## Архитектура

```
corrector.avar.me
├── corrections.json   ← правила коррекции (хранятся в репозитории)
├── build.js           ← скрипт сборки (выкачивает CSV, генерирует dist/)
└── src/               ← статический фронтенд
```

Корпус частотностей лемм **не хранится в репозитории** — при каждой сборке выкачивается из:
```
https://forms.avar.me/data/lemma_frequencies.csv
```

## Правки коррекций

Все правила хранятся в `corrections.json`:

- **`normalize`** — нормализация ввода (замена псевдосимволов палочки: г1 → гӏ, гI → гӏ, гl → гӏ и т.д.), применяется молча
- **`ocr`** — OCR-ошибки (применяются только в текстовом режиме с включённым «OCR-режим»)
- **`letters`** — обязательные замены букв/буквосочетаний (например, пӏ → б); применяются всегда и показываются как исправление
- **`words`** — обязательные замены целых слов (например, дий → дие); применяются всегда и показываются как исправление

## Алгоритм

1. Нормализация слова (правила `normalize`)
2. Точный поиск в словаре
3. Генерация кандидатов на расстоянии 1 (удаление, транспозиция, замена, вставка)
4. Если кандидатов < 3 — генерация кандидатов на расстоянии 2
5. Скоринг: `score = freq / (50 ^ (dist - 1))`
6. Нормализация вероятностей

## Сборка и деплой

```bash
node build.js   # генерирует dist/
```

GitHub Actions деплоит `dist/` на ветку `gh-pages` при каждом пуше в `main`.
